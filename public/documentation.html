myFlix

myFlix is the web application which provides users with access to information about different movies, directors, and genres. Users are able to sign up, update their personal information, and create a list of their favorite movies.
myFlix consists of well designed REST API and architected database built using JavaScript, Node.js, Express, and MongoDB.
The server­side of the myFlix application processes user requests & performs operations against the data stored in the database.

Description URL HTTP method Request body data format Response body data format
Get a list of all movies to the user /movies GET None JSON object holding data about all movies. Example:

{ Title: " The Godfather", description: “movie description”, Genre: {Name: "Drama", Description: " Movie with serious undertones."}, Director: { Francis Ford Cappola", Bio: "Full directors bio", Birth: "1939", Death: "2018"}, Actors: ["Marlon Brando", "Al Pacino"], ImagePath: "godfather.png", Featured: true}

Get data about a single movie by title /movies/[title] GET None JSON object holding data about a single movie. Example:

{ Title: " The Godfather", description: “movie description”, Genre: {Name: "Drama", Description: " Movie with serious undertones."}, Director: { Francis Ford Cappola", Bio: "Full directors bio", Birth: "1939", Death: "2018"}, Actors: ["Marlon Brando", "Al Pacino"], ImagePath: "godfather.png", Featured: true}

Get data about a genre by name/title /genre/[name] GET None A JSON object holding data about a genre (description) by name/title. Example:

{Name: "Drama", Description: "Movie with serious undertones, etc."}

Return data about a director by name /director/[name] GET None A JSON object holding data about a director (bio, birth year, death year) by name. Example:

{ Name: ‘”Francis Ford Cappola” bio: ” ..full bio..”, birth year: "1939", death year: “2018” }

Add a new user to the registry. /users POST JSON object holding data about the user to add. Example:

{ username: "John Smith", password: "Malipiso123", email: "john.smith@email.com", dob: "01/01/1988"}

JSON object holding data about the user that was add. Example:

{id: 3, username: "John Smith", password: "Malipiso123", email: "john.smith@email.com", dob: "01/01/1988"}

Update a user's information /users/[username] PUT JSON object holding data about the user to add. Example:

{ username: "John Smith", password: "Malipiso123", dob: "01/01/1988"}

Confirmation message advising that user's information has been updated.
Allow users to add a movie to their list of favorites /favorites/[username]/movies/[movieID] POST None Confirmation message advising that movie has been added to the favourite list.
Allow users to remove a movie from their list of favorites /favorites/[username]/movies/[movieID] DELETE None Message advising that movie has been deleted from the favourite list.
Allow existing users to deregister /users/[username] DELETE None Message advising that user has been removed from the registry.
